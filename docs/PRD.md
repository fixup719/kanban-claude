# PRD — Kanban Board

## 1. 제품 개요

**제품명:** Kanban Board  
**버전:** 1.0.0  
**작성일:** 2026-06-18  
**작성자:** fixup719

간단하고 직관적인 칸반 보드 웹 애플리케이션이다. 개인 또는 소규모 팀이 할 일, 진행 중, 완료 상태로 작업을 시각적으로 관리할 수 있도록 한다.

---

## 2. 배경 및 목표

### 배경
기존 스프레드시트나 메모 앱은 작업의 상태 흐름을 시각화하기 어렵다. 드래그 앤 드롭 기반 칸반 보드를 통해 작업 흐름을 직관적으로 파악하고 관리 효율을 높인다.

### 목표

| # | 목표 | 성공 지표 |
|---|------|-----------|
| 1 | 작업 상태를 시각적으로 관리 | 3개 컬럼(TO-DO / IN-PROGRESS / DONE) 구현 |
| 2 | 카드 생성·삭제·이동 | 드래그 앤 드롭 + 모달 입력 폼 |
| 3 | 우선순위 표시 | 높음 / 보통 / 낮음 3단계 배지 |
| 4 | 빠른 초기 로드 | 외부 라이브러리 없이 순수 HTML/CSS/JS |

---

## 3. 대상 사용자

- **개인 개발자 / 학생** — 사이드 프로젝트 또는 과제 일정 관리
- **소규모 팀 (2~5명)** — 스프린트 없이 간단한 작업 분배 필요
- **업무 자동화 입문자** — 복잡한 툴(Jira, Notion) 없이 칸반 개념 학습

## 4. 기능 요구사항

### 4.1 핵심 기능 (MVP)

#### FR-01 컬럼 표시
- 보드는 **TO-DO**, **IN-PROGRESS**, **DONE** 세 컬럼을 항상 표시한다.
- 각 컬럼 헤더에 현재 카드 수를 뱃지로 표시한다.

#### FR-02 카드 추가
- 헤더의 `+ 카드 추가` 버튼 클릭 시 모달이 열린다.
- 모달 입력 필드: **제목** (필수, 최대 50자), **설명** (선택, 최대 200자), **우선순위** (낮음 / 보통 / 높음, 기본: 보통)
- 추가된 카드는 **TO-DO** 컬럼 맨 아래에 삽입된다.
- 제목 미입력 시 저장 불가 (입력창 붉은 테두리 표시).
- Enter 키로 확인, Escape 키로 취소한다.

#### FR-03 카드 삭제
- 카드 우측 하단 `✕` 버튼 클릭 시 즉시 삭제된다.
- 삭제 확인 다이얼로그는 제공하지 않는다 (빠른 조작 우선).

#### FR-04 드래그 앤 드롭 이동
- 카드를 다른 컬럼으로 드래그하여 상태를 변경한다.
- 같은 컬럼 내 순서 변경도 지원한다.
- 드래그 중 대상 컬럼에 드롭 플레이스홀더(점선 박스)를 표시한다.
- 드래그 중 카드는 반투명(opacity 0.4)으로 표시한다.

#### FR-05 우선순위 표시
- 카드 좌측 보더 색상으로 우선순위를 구분한다.
  - 높음: `#e94560` (빨강)
  - 보통: `#f0a500` (주황)
  - 낮음: `#66bb6a` (초록)
- 카드 하단에 우선순위 배지 텍스트(높음/보통/낮음)를 표시한다.

### 4.2 비기능 요구사항

| 구분 | 요구사항 |
|------|---------|
| 성능 | 초기 페이지 로드 1초 이내 (외부 CDN 없음) |
| 접근성 | 키보드 단축키(Enter/Escape) 지원 |
| 반응형 | 모바일(768px 이하) 세로 컬럼 레이아웃 |
| 보안 | XSS 방지 — 모든 사용자 입력 `escapeHtml` 처리 |
| 브라우저 | Chrome / Edge / Firefox 최신 버전 지원 |
