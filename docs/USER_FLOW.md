# User Flow — Kanban Board

## 1. 전체 사용자 흐름 개요

```
앱 진입 (index.html 로드)
        │
        ▼
보드 화면 표시
(TO-DO / IN-PROGRESS / DONE 컬럼)
        │
   ┌────┴─────────────┐
   │                  │
   ▼                  ▼
카드 추가           기존 카드 조작
   │              ┌───┴───────────┐
   │              │               │
   │           드래그          카드 삭제
   │           앤 드롭
   │              │
   └──────────────┘
        │
        ▼
  상태 업데이트 & 화면 재렌더
```

---

## 2. 플로우 1 — 카드 추가

```
[헤더] "+ 카드 추가" 버튼 클릭
        │
        ▼
모달 오픈 (배경 블러 + 오버레이)
        │
        ▼
사용자 입력
  ├─ 제목 입력 (필수, 최대 50자)
  ├─ 설명 입력 (선택, 최대 200자)
  └─ 우선순위 선택 (낮음 / 보통[기본] / 높음)
        │
   ┌────┴──────────────────────────┐
   │                               │
"추가" 버튼 클릭                취소 / Esc / 오버레이 클릭
또는 Enter 키                          │
   │                               ▼
   ▼                          모달 닫기 (입력 초기화)
제목 비어있음?
   │
 YES ──→ 입력창 빨간 테두리 표시 → 포커스 유지 (흐름 중단)
   │
  NO
   │
   ▼
cards[] 배열에 카드 객체 추가
(status: 'todo', id: nextId++)
        │
        ▼
모달 닫기 + 입력 초기화
        │
        ▼
render() → TO-DO 컬럼 맨 아래에 새 카드 표시
        │
        ▼
컬럼 카드 수 배지 갱신
```

---

## 3. 플로우 2 — 드래그 앤 드롭으로 카드 이동

```
카드에 마우스 버튼 누름
        │
        ▼
dragstart 이벤트 발생
  - draggedCard 변수에 카드 객체 저장
  - 카드 DOM에 .dragging 클래스 추가 (반투명)
        │
        ▼
마우스를 다른 컬럼 위로 이동
        │
        ▼
dragover 이벤트 (대상 컬럼)
  - 컬럼에 .drag-over 클래스 추가 (테두리 하이라이트)
  - 드롭 플레이스홀더(점선 박스) 생성 및 위치 삽입
  - getDragAfterElement()로 마우스 Y 기준 삽입 위치 계산
        │
   컬럼 밖으로 이동?
   YES ──→ dragleave 이벤트
         - .drag-over 클래스 제거
         - 플레이스홀더 제거
        │
  NO (컬럼 위에서 드롭)
        │
        ▼
drop 이벤트 발생
  - 플레이스홀더 제거
  - .drag-over 클래스 제거
  - cardObj.status = 새 컬럼 status 값
  - cards[] 배열에서 카드 제거 후 목표 위치에 splice 삽입
        │
        ▼
dragend 이벤트
  - .dragging 클래스 제거
  - draggedCard = null
        │
        ▼
render() → 전체 보드 재렌더
  - 원래 컬럼 카드 수 감소
  - 대상 컬럼 카드 수 증가
```

---

## 4. 플로우 3 — 카드 삭제

```
카드 우측 하단 "✕" 버튼 클릭
        │
        ▼
click 이벤트 (e.stopPropagation으로 드래그 이벤트와 충돌 방지)
        │
        ▼
deleteCard(id) 호출
  - cards[] = cards.filter(c => c.id !== id)
        │
        ▼
render() → 해당 카드 제거 후 재렌더
        │
        ▼
컬럼 카드 수 배지 갱신
```

---

## 5. 키보드 단축키

| 키 | 동작 |
|----|------|
| `Enter` | 모달이 열려 있을 때 → 카드 추가 확인 |
| `Escape` | 모달이 열려 있을 때 → 모달 닫기 |

---

## 6. 상태 전이 다이어그램

```
          + 카드 추가                드래그 →               드래그 →
              │                     IN-PROGRESS             DONE
              ▼                        │                      │
          [ TO-DO ] ──────────────► [ IN-PROGRESS ] ──────► [ DONE ]
              ▲                        │                      │
              │         드래그 ←       │      드래그 ←        │
              └────────────────────────┘◄─────────────────────┘

  * 모든 컬럼 간 양방향 이동 가능
  * 같은 컬럼 내 순서 변경 가능
```

---

## 7. 에러 / 엣지 케이스

| 상황 | 처리 방식 |
|------|-----------|
| 제목 없이 카드 추가 시도 | 입력창 빨간 테두리, 포커스 유지, 추가 중단 |
| 드래그 중 드롭 대상 없음 | `dragend`에서 `draggedCard = null` 초기화 |
| 동일 컬럼 내 드롭 | 순서만 변경, status는 동일하게 유지 |
| 모달 외부(오버레이) 클릭 | 모달 닫기, 입력 초기화 |
